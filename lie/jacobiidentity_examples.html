  2. Evaluate the following code.

    This tells you that there are several problems which prevent this multiplication table from defining a 3-dimensional Lie algebra.

    • The entry [3,3,2,-9] corresponds to the structure constant \(c_{33}^2=-9\) and violates the antisymmetry, so it is simply ignored by our algorithm.

    • The entry [2,1,3,5] corresponds to the structure constant \(c_{21}^3=5\text{.}\) Our algorithm enforces antisymmetry, which means that it also implies that \(c_{12}^3=-5\text{.}\) But there is a conflict with the entry [1,2,3] which is interpreted as \(c_{12}^3=-c_{21}^3=1\text{.}\) As a result, our algorithm ends up ignoring the entry [2,1,3,5].

    • On top of this, there is a problem with the Jacobi identity equation (1.1.2) corresponding to \(i=1,j=2,k=3,m=3\text{,}\) namely

      \begin{equation*} \sum_{l=1}^3c_{1l}^3c_{23}^l+c_{2l}^3c_{31}^l+c_{3l}^3c_{12}^l\ne 0. \end{equation*}

      A nontrivial way to fix this is to add the nontrivial structure constant \(c_{23}^2=-c_{32}^2=-7\text{,}\) as you can see by evaluating the following code.

  3. Let us consider the parameterized family of 6-dimensional Lie algebras \(L_{6,19}(t)\) with nonzero brackets

    \begin{equation*} [x_1,x_2]=x_4,\quad [x_1,x_3]=x_5,\quad [x_2,x_4]=x_6,\quad [x_3,x_5]=tx_6 \end{equation*}

    and store it in a dictionary containing the base field, the dimension, the table, and the structure constants. Notice that we choose to see it as a 6-dimensional algebra over the rational function field \(\mathbb{Q}(t)\text{.}\)
